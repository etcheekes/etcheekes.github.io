Lift-It

Try Lift-It here.

Lift-It is a weightlifting-focused web application with a user-friendly interface that helps users create personalised workout routines:

It has several key features, including:

  • Browse 206 default exercises by name, muscle group, or equipment required.
  • Add or delete exercises from the available database.
  • Create personalised workout routines by selecting exercises from the app's database and choosing the number of reps and weight for each exercise.
  • Dynamically modify created routines by adjusting the number of repetitions, weight, or adding/removing exercises.
  • Account creation allows users to browse their unique set of exercises and store their workout routines for access during subsequent app visits.

Click below if you're interested in learning about the technical inner-workings of Lift-It or how my technical experience was developed through completing this app.

Lessons learned

As a full-stack application, creating Lift-It provided valuable learning opportunities. Some of the key areas of growth include:

  • An improved higher-level understanding of the roles that frontend, backend, and databases play and how they interact with each other to create a functional web application.
  • Greater understanding of how static and dynamic elements can be used in a web application.
  • Experience in implementing HTML, CSS, vanilla JavaScript, Python, two Python backend frameworks (Flask and Django), SQL and database management (SQLite and PostgreSQL) to create a web application.
  • Experience deploying a web application using a PaaS (platform as a service).
  • Appreciation for the importance of design and UI.
  • Experience implementing user management, including user authentication, sessions, and cookies.
  • Experience in applying version control using Git and GitHub.

Technologies used

Frontend: HTML5 and CSS (vanilla and Bootstrap framework) created the basic structure of the web application. JavaScript and its Fetch API were used to implement dynamic elements.

Backend: I used Flask and Django, two Python web application frameworks, for the first and second iterations of Lift-It respectively. In each iteration the frameworks’ templating engines were used to render dynamic content with predefined HTML templates based on user input and data changes.

Database Management: Database Management: The first iteration of Lift-It used SQLite while the second used PostgreSQL as its database management systems. This involved defining the database schema and using SQL (for Flask) or Django’s Object-Relational Mapping (ORM) system for querying the database.
